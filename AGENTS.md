# LiteLLM Docker Proxy — Agent Guide

LLM gateway that routes OpenCode / Claude Code requests through a local
`opencode serve` instance. All model calls flow:

```
client → LiteLLM (:4000) → proxy sidecar (:8080) → opencode serve (:4096) → upstream provider
```

---

## Key files

| File | Purpose |
|------|---------|
| `Makefile` | All operational commands — start here |
| `proxy.py` | Translates OpenAI/Anthropic wire format → OpenCode session API |
| `config.local.yaml` | LiteLLM model/routing config (not in git) |
| `config.tunnel.yaml` | Same as above but for tunnel-exposed deployments |
| `docker-compose.yml` | Defines litellm, proxy, postgres, cloudflared containers |
| `.env` | Runtime secrets — generated by `make init`, never committed |
| `.env.example` | Template used by `make init` |

---

## Secrets

All secrets live in `.env`. Generate them with:

```bash
make init
```

This writes random hex values for:
- `LITELLM_MASTER_KEY` — the API key every client must send
- `LITELLM_SALT_KEY` — internal LiteLLM signing key
- `UI_USERNAME` / `UI_PASSWORD` — Web UI login at `/ui/`

To retrieve the master key at any time:
```bash
grep LITELLM_MASTER_KEY .env | cut -d= -f2
```

## Setup scenarios

### A — Local only (no public access)

Everything runs on `localhost`. Use this for development.

```bash
make init             # generate .env + cacert.pem (first time only)
make auth             # authenticate opencode serve — opens browser (first time only)
make local            # start opencode serve + LiteLLM + proxy, no tunnel
make inject-opencode  # write LiteLLM provider into ~/.config/opencode/opencode.json
```

Configure Claude Code:
```bash
export ANTHROPIC_BASE_URL=http://localhost:4000/v1
export ANTHROPIC_API_KEY=$(grep LITELLM_MASTER_KEY .env | cut -d= -f2)
```

Verify:
```bash
make health
make test-curl
```

---

### B — Tunnel (public access via Cloudflare)

Exposes LiteLLM at `https://litellm.ryan-schachte.com` through a Cloudflare Tunnel.

```bash
make init                                                          # first time only
make auth                                                          # first time only
make up                                                            # starts cloudflared too
make inject-opencode LITELLM_URL=https://litellm.ryan-schachte.com/v1
```

Configure Claude Code:
```bash
export ANTHROPIC_BASE_URL=https://litellm.ryan-schachte.com/v1
export ANTHROPIC_API_KEY=$(grep LITELLM_MASTER_KEY .env | cut -d= -f2)
```

On a **remote machine** that doesn't run the stack, authenticate cloudflared once:
```bash
brew install cloudflared
cloudflared access login https://litellm.ryan-schachte.com
```

---

### C — Remote-only (stack already running elsewhere)

No docker needed. Just configure the clients to point at the running tunnel.
Get the `LITELLM_MASTER_KEY` from whoever runs the stack.

**OpenCode** — inject provider config (reads `config.local.yaml` for model list):
```bash
LITELLM_MASTER_KEY=<key> make inject-opencode LITELLM_URL=https://litellm.ryan-schachte.com/v1
export LITELLM_MASTER_KEY=<key>
```

Then in OpenCode, run `/models` and select any model under **LiteLLM Proxy**.

**Claude Code:**
```bash
export ANTHROPIC_BASE_URL=https://litellm.ryan-schachte.com/v1
export ANTHROPIC_API_KEY=<key>
```

**OpenAI-compatible clients** (Cursor, Windsurf, etc.):
```
Base URL : https://litellm.ryan-schachte.com/v1
API Key  : <LITELLM_MASTER_KEY>
```

## inject-opencode detail

`make inject-opencode` parses `config.local.yaml` at runtime to pick up all
`model_name` entries and merges the `litellm` provider block into
`~/.config/opencode/opencode.json`. Existing keys in the file are preserved.

```bash
make inject-opencode                                        # → http://localhost:4000/v1
make inject-opencode LITELLM_URL=https://litellm.ryan-schachte.com/v1  # → tunnel
```

Re-run any time models are added to `config.local.yaml`.

## Adding or modifying models

Edit `config.local.yaml`. Each entry follows this pattern:

```yaml
- model_name: my-new-model          # the ID clients use
  litellm_params:
    model: anthropic/my-new-model   # provider/model for LiteLLM routing
    api_base: http://proxy:8080     # always routes through the proxy sidecar
    api_key: not-used
```

Provider prefixes used in `model:`:
- `anthropic/` — routes through `/v1/messages` handler in proxy.py
- `openai/` — routes through `/v1/chat/completions` handler in proxy.py

After editing, re-run `make inject-opencode` to sync the opencode.json model list.

## Common commands

```
make init             First-time setup: generate .env + cacert.pem
make auth             Authenticate opencode serve (opens browser)
make up               Start full stack with Cloudflare Tunnel
make local            Start full stack, localhost only
make down             Stop everything including opencode serve
make restart          Restart all containers
make inject-opencode  Write LiteLLM provider into ~/.config/opencode/opencode.json
make logs             Tail all docker logs
make logs-proxy       Tail proxy sidecar logs
make logs-litellm     Tail LiteLLM logs
make logs-serve       Tail opencode serve logs
make status           Show container status
make health           Check health endpoints
make test-curl        Smoke test against localhost:4000
```
